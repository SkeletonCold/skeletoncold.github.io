🏠开始

本教程只适用于Java版
需要安装Java并配置环境变量
https://java.com/环境变量，请自行百度如何配置Java环境变量
请打开文件扩展名显示
下文中所有图文bat程序改为bat脚本毕竟只是个启动脚本...叫bat脚本更妥当

🚀搭建

  • 1 🍀原版服务端

  • 1.1 下载服务端

    • 1.11首先访问网址https://minecraft-zh.gamepedia.com/Java%E7%89%88%E7%89%88%E6%9C%AC%E8%AE%B0%E5%BD%95
    • 1.12然后选择一个你想要的版本，本教程以1.16.3版本为例
    • 1.13进入页面后选择服务端下载

      下载后我们会得到一个server.jar的文件，把他放到一个你记得住的位置，可以将他重命名一下方便辨认server1.16.3.jar什么！你说你下载到的是server，没有任何后缀名？那么请打开显示扩展名https://jingyan.baidu.com/article/f00622282564bdfbd3f0c827.html注意！该步骤很关键！后续操作也需要更改扩展名之类的

  • 1.2 搭建服务端

    • 正常情况下（指配置好环境变量）能直接打开服务端，但是为了更多自定义选项，更好的管理请使用dos启动服务端
    • 1.21在server.jar的同级目录下创建一个新建文本文档
    • 1.22然后将其后缀改为bat前缀名字无所谓，弹出的提示框，选择是
    • 1.23右键选择编辑或用其他文本编辑器打开
    • 1.24打开后，输入java -Xmx6144M -Xms1024M -jar server.jar nogui下图是各项参数说明，编辑好后，点击左上角【文件】->【保存】或快捷键Ctrl+s【没用过Mac，mac应该是command+s】

      补:最后的nogui是在运行的时候不显示java程序的gui，只显示dos窗口
    • 1.25编辑好后保存，然后打开bat程序
      一段时间后，bat会启动服务端文件进行初构建，并关闭掉dos窗口，然后会生成以下文件

      补：图中“服务端配置”->“服务器配置”
    • 1.26打开eula.txt将eula=false改为eula=true编辑完后保存文件
    • 1.27再次打开bat文件，服务端将开始工作（生成世界巴拉巴拉的）最下方显示Done说明服务端启动成功了！这是初构建，所以启动可能较慢

  • 3 🔧Forge/Fabric服务端

因为差不多，所以打算并到一起讲= =，其实上面插件端和原版端也是差不多的

  • 3.1 下载服务端

  • Forgehttp://files.minecraftforge.net/maven/net/minecraftforge/forge

    在左侧选择你想要的版本，然后选择红框里的推荐版（左侧是最新版，可能有bug！）在红框里找到installer点击进行下载，注:这是链接到adfoc.us的广告链接，观看广告，点击跳过键才能下载，国内无法加载出skip键，则无法下载！
    下载好后放在一个你能记住的地方
  • Fabrichttps://fabricmc.net/

    打开链接后，先点击上方use然后选择一个文件进行下载，这两个文件都是一样的！只是一个更方便windows打开，一个是全平台的，我们这边下载jar版的
    下载好后放在一个你能记住的地方

  • 3.2 安装器处理

  • Fabric

    • fabric下载下来是个安装器，并不是服务端文件，所以我们先要对他进行一步处理
    • 在fabric-installer-版本号.jar的同级目录下创建一个新建文本文档
    • 然后将其后缀改为bat前缀名字无所谓，弹出的提示框，选择是什么！你说你文件后面没有任何后缀名？那么请打开显示扩展名https://jingyan.baidu.com/article/f00622282564bdfbd3f0c827.html
    • 右键选择编辑或用其他文本编辑器打开
    • 打开后，输入java -jar fabric-installer-0.6.1.51.jar编辑好后，点击左上角【文件】->【保存】或快捷键Ctrl+s【没用过Mac，mac应该是command+s】
    • 编辑好后运行bat
    • 他看起来是这样的
    • 选择服务端，然后选择你想要的游戏版本，文件目录，然后点击安装
    • 安装完之后，fabric会提供服务器启动脚本即上面原版/插件端的bat程序，以及原版服务端下载

      注:原版服务端是必下的！fabric/forge本身不提供游戏内容，他只是个框架API，所以需要原生游戏的支持
      安装完后关闭那2个窗口，如果看那安装器不爽，可以把他删掉
  • 好！到此为止，fabric服务端安装整完了

  • Forge

    • 和fabric差不多，所以你会的话可以不往下看
    • 在forge-游戏版本-forge版本-installer.jar的同级目录下创建一个新建文本文档
    • 然后将其后缀改为bat前缀名字无所谓，弹出的提示框，选择是什么！你说你文件后面没有任何后缀名？那么请打开显示扩展名https://jingyan.baidu.com/article/f00622282564bdfbd3f0c827.html
    • 右键选择编辑或用其他文本编辑器打开
    • 打开后，输入java -jar forge-1.16.3-34.1.0-installer.jar编辑好后，点击左上角【文件】->【保存】或快捷键Ctrl+s【没用过Mac，mac应该是command+s】
    • 编辑好后运行bat
    • 他看起来是这样的
    • 选择中间的Install server然后点击右侧的3个点，选择路径，再点击确定开始安装，弹出下图的提示，说明安装完了！
    • 文件目录一览如果看那安装器不爽，可以把他删掉

⚙️配置

服务器相关配置

所有服务端都是围绕原版服务端展开的所以不管你是什么服务端，都会有这么几个文件

服务器配置文件server.properties里的内容请参阅wiki，没有服务端可以添加里面的设置项，所以参阅wiki不会有差错，而且版本不同，里面的内容也不同，比如说老版本的没xxx配置https://minecraft-zh.gamepedia.com/Server.properties翻到下面，找到Minecraft服务器属性-Java版

服务器插件相关配置

安装插件

想要安装插件，必须使用插件端！你想既用插件，又用模组？？Sponge Forge可以满足你的需求！不过Sponge Forge最高只支持1.12.2
有了插件端，那么就是去哪下载插件了emmm，这个怎么说呢，得你自己去那个插件端的官网，然后找一下链接什么的....就说几个吧https://www.spigot.org``https://forums.spongepowered.org/别的我就不知道了= =，自己找吧....
好！现在你找到了插件，并下载了下来，那么安装在哪？？把下载的插件丢到服务端的plugins文件夹下（需启动一次服务器才会生成这个文件夹），然后启动服务端，即可装载插件，如果你的服务器在运行，也是可以热添加插件的，不过要输入/reload指令注：/reload在高版本才有，而且经常会出现不兼容的情况！比如说使用了reload指令后服务器卡死，这时需要强制结束服务端才有效

配置插件

插件下载好了那肯定得配置啊，那该如何配置呢，将插件丢到plugins文件夹后启动一次服务器，如果这个插件他有配置文件，则会自动生成一个配置文件在plugins文件夹下，他看起来是这样的

关于如何配置xx插件，请自行翻阅该插件的wiki！

服务器模组相关配置

安装模组

想要安装模组，必须使用fabric/forge端，并启动一次服务器，随后会在服务端目录下生成一个mods文件夹这里面就是给你放模组的！
模组下载地址https://www.curseforge.com/minecraft/mc-modsemmm最常用的就是这个了= =

配置模组

把模组放入mods文件夹后，启动一次服务器，如果这个模组他有配置文件，则会自动生成一个配置文件在服务端的config文件夹下，至于服务端的defaultconfig文件夹....我也不知道干啥用的，没见哪个模组把文件生成在这

💽指令

emmm也没啥好说的吧，你说模组的指令，你又不一定用这模组，插件也是，就讲讲原版的几个吧
/stop停止服务器，各种服务器都可以用这个指令停止！如果服务器卡死了，输入指令无效的话，那么请自行结束服务端进程（该指令需要4级权限才能使用，服务器配置文件（server.properties）里默认给OP的就是4级权限，所以OP不规范，服主两行泪）
/op <id>该指令可以给玩家OP权限，可以在**服务器配置文件（server.properties）**里设置默认给玩家的是几级权限
/deop <id>该指令可以取消玩家的OP权限
